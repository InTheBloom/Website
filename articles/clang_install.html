はじめに

このページは、Windowsを使っている人で、「C言語を使ったプログラミングをしてみたいけど始め方がよくわからない」と思っている人に向けたページです。

このページで紹介するのはLLVM/Clang + VisualStudio Build Toolsを使用したC言語処理系のセットアップの流れです。現状C言語の環境構築に関する情報は、新しいものから古いものまで錯綜しており、初学者にとっては負担になると言わざるを得ません。そこで、このページは私が実際にWindows11上でC言語のコンパイルをできるようになるまでの過程を紹介しつつ、迷える初学者に有用であると思われる情報への足掛かりを提供します。

注意: 私もC言語の学習を始めて数か月の初心者ですから、私自身が発信する情報に関しては単に「こうやったらできます」程度に受け取ってください。私が知りえる範囲で有用だと思う情報はこのページ内にリンクとして載せてありますから、より正確で詳しい情報はそちらを参照してください。

C言語を始めたい！でも何すればいいの？

この章は本題と関係ないことを結構書きますので、飛ばしていただいても影響はほぼありません。

1972年にC言語が開発されてから早50年。未だにCは人気のプログラミング言語です。しかしその人気とは裏腹に、初めてのプログラミングとしてC言語を始めようとすると、様々な困難に見舞われるかもしれません。私見では、このような状況が起こるのは現代の非常に抽象化されたコンピューターシステムが、ユーザーにほとんど知識を求めないからであると思っています。例えばMicrosoft Office製品で文書などを作成するところを想像してみましょう。あなたがするべきことは、

  1. Officeを自分のコンピューターにインストールする。
  2. デスクトップなどに作られるショートカットをダブルクリックする。
  3. 視覚的直観に任せて操作をする。
  4. ファイルを保存する

といったところでしょうか。この時、ユーザーはコマンドラインインターフェースに触れることはおろか、コンピューターのファイルシステムを意識することなくファイルの保存を行うことができます。次に同じファイルを開くときも、きっとソフトウェアが親切に以前開いたファイルを案内に出してくれるでしょう。

一方、ソフトウェアを生み出す立場になったらどうでしょうか？きっと今までいかにソフトウェアに守られながらコンピューターを使っていたのかを思い知らされることになることでしょう。(実際私がそうでした。)CLIで簡単なコマンドが操れることはもはや前提といってもよく、さらにその下にある階層型のファイルシステムや、カレントディレクトリなどの概念などは公式のリファレンスに載っているはずもありません。

このように、単にコンピューターを利用するだけの人と、コンピューターを利用して新しいことをしようとする人で必要になる知識に大幅な差が生まれているといえるでしょう。このことから、C言語を始めようとする中で環境構築が分からなくなってしまう人は理解している人との前提の共有ができていないことが多いと考えています。このページでは方法論としてC処理系のセットアップを紹介しますが、単に「こうしたらできた」で済ませるのではなく、その「こうしたら」などの部分の必然性などをある程度追いかけて知っておくということも上達を助けるのではないでしょうか？
