現在のディレクトリ構造

11/23 ~ ディレクトリ構造は以下のようになっています。rootはhttps://inthebloom.github.io/です。

root -
     | - home.html
     | - favicon.png
     | - default.css
     |
     | - articles
     | | - 〇〇.html
     | | - (同上)
     |
     | - pictures
     | |
     | | - (一つの.htmlファイルに対応するディレクトリ)
     | | | - (メディア)
     | |
     | | -(同上)
     |
     | - README.md
     | - templete.html

記事作成の流れ

  1. root直下にあるtemplete.htmlを使用して，articlesディレクトリ内に半角英語でファイルを作成しします。
  2. 作成したファイルのhead内の必要な事項をすべて埋めます。具体的には，
    • タイトル
    • 説明(metaタグのdescription)
    • ogpのurl
    • ogpのdescription
    • ogpのイメージパス
    です。
  3. 記事本体を書きます。
  4. root直下にあるpicturesディレクトリ内にhtmlファイルと同じ名前のディレクトリを作って，記事本体で使用したメディアなどを入れます。さらに，ogp用のイメージを1280×720の解像度で作成して，ogp.pngという名前にして入れておきます。
  5. home.htmlに新しい記事へのリンクを追加します。(忘れやすいので注意してください)

cssの注意点

現在このウェブサイトのすべてのページは，一括してroot直下にあるdefault.cssによって装飾されています。このcssに関して憶えておくべきことを書いておきます。

cssによって装飾される要素のリスト

2022/12/14現在、最新の状況を反映できていません。

  • pre要素 > overflow: scroll;が適用されています。主にプログラムを載せるときに，形を崩さずに，かつモバイルデバイスでの可視性向上を図っています。
  • img要素 > max-width: 100%; height: auto; border: 1px solid black;が適用されています。画面の横幅が画像のサイズに対して小さい時に自動的に横幅に合わせるようになっています。
  • blockquote要素 > margin: 0;が適用されています。なんか勝手に字下げしまくるのでマージン消しました。
  • (blockquote要素内の)p要素 > padding: 15px; background: #eee; border-radius: 5px;が適用されています。単純に見栄えをよくするだけです。
  • h2要素 > padding-bottom: 10px; border-bottom: 1px solid black;が適用されています。セクションの切り替わりに使うために，cssで下線を引くようにしています。

htmlのややこしいもの

よく使うだろうけど間違えそうなhtmlのお手本を載せておきます。

2022/12/14現在、最新の状況を反映できていません。

<figure>
